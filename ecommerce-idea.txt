# Important Development Rules

1. Git Workflow Rules

Each phase MUST:

* Be developed in a separate Git branch
* Be committed properly
* Be merged into `main` branch after testing
* Include at least one intentional minor merge conflict demonstration

# Git Workflow Instructions

For every phase:

git checkout development
git pull origin development

git checkout -b feature/module-name

After completion:

git add .
git commit -m "Completed Authentication Module"

git push origin feature/module-name

Merge process:

git checkout development
git merge feature/module-name

Final merge:

git checkout main
git merge development

# Merge Conflict Demonstration Requirement

During at least 2 modules:

* Modify same HTML file in two branches
* Modify same CSS file in two branches
* Create intentional merge conflict
* Resolve conflict manually
* Commit resolved version

Example:

CONFLICT (content): Merge conflict in templates/base.html

Resolve manually and commit:

git add .
git commit -m "Resolved merge conflict in base.html"

# UUID Requirements

The following entities MUST use UUID instead of integer IDs.

UUID Entities

| Entity        | UUID Required |
| Customer      | YES           |
| Seller/Vendor | YES           |
| Invoice       | YES           |
| Orders        | YES           |

UUID Example

python
import uuid

customer_id = models.UUIDField(
    primary_key=True,
    default=uuid.uuid4,
    editable=False
)

# SKU Requirements

Every product MUST have a unique SKU.

Format:

text
ELEC-0001
BOOK-0002
FASH-0003

SKU Rules:

* Auto-generated
* Unique
* Category-based prefix
* Immutable after creation

# Suggested SKU Generator Logic

Example:

python
category_prefix = "ELEC"
product_count = Product.objects.count() + 1

sku = f"{category_prefix}-{product_count:04d}"
